# Creator Tool — Config Reference

This document describes every `application.yaml` key that world builders and balance
designers can tune without writing Kotlin code. It is written for the **AmbonMUD Creator
Tool** (AmbonMUD-Visualize successor) and the Arcanum world-building team.

All values shown are the shipping defaults. Omitting a key from your YAML keeps the
default. All keys live under the `ambonmud:` root.

## Character Creation

Path: `ambonmud.engine.characterCreation`

Controls what a brand-new character starts with the moment they complete account creation.

```yaml
ambonmud:
  engine:
    characterCreation:
      startingGold: 0   # gold placed in the player's purse at creation (integer >= 0)
```

| Key | Default | Notes |
|-----|---------|-------|
| `startingGold` | `0` | One-time gold grant. Does not affect existing characters. Set to a small value (e.g. `25`) to let new players buy one basic item from a starter shop. |

## Player Classes

Path: `ambonmud.engine.classes.definitions`

Each key is the class ID (uppercase). IDs are case-insensitive at runtime.

```yaml
ambonmud:
  engine:
    classes:
      definitions:
        WARRIOR:
          displayName: "Warrior"
          description: "A stalwart melee fighter."
          hpPerLevel: 8           # HP gained per level (stacks with base HP)
          manaPerLevel: 4         # Mana gained per level
          primaryStat: STR        # UI hint for the stat sheet; does not affect gameplay
          selectable: true        # false = hidden from character creation (debug/NPC classes)
          startRoom: ""           # optional RoomId override for new characters of this class
          threatMultiplier: 1.5   # combat threat multiplier (1.0 = neutral, > 1.0 = tank)
        MAGE:
          displayName: "Mage"
          hpPerLevel: 4
          manaPerLevel: 16
          primaryStat: INT
        CLERIC:
          displayName: "Cleric"
          hpPerLevel: 6
          manaPerLevel: 12
          primaryStat: WIS
        ROGUE:
          displayName: "Rogue"
          hpPerLevel: 5
          manaPerLevel: 8
          primaryStat: DEX
        SWARM:
          displayName: "Swarm"
          hpPerLevel: 2
          manaPerLevel: 3
          selectable: false
```

### Class Fields

| Field | Default | Notes |
|-------|---------|-------|
| `displayName` | (key name) | Shown in character creation and score sheet. |
| `description` | `""` | Shown in the character creation class-selection prompt. |
| `hpPerLevel` | `4` | HP added each time the character gains a level. Stacks with CON scaling. |
| `manaPerLevel` | `8` | Mana added each level. Stacks with INT scaling. |
| `primaryStat` | `""` | Informational stat ID displayed in the UI. No mechanical effect. |
| `selectable` | `true` | When `false`, the class is hidden from new-character creation. |
| `startRoom` | `""` | Fully-qualified `zone:room` ID. When set, new characters of this class spawn here instead of the world start room. |
| `threatMultiplier` | `1.0` | Multiplies all melee threat generated by this class. Values above 1.0 make mobs prefer targeting this class (tank role). Values below 1.0 make mobs de-prioritise this class (glass cannon). **WARRIOR defaults to 1.5.** |

## Races

Path: `ambonmud.engine.races.definitions`

```yaml
ambonmud:
  engine:
    races:
      definitions:
        HUMAN:
          displayName: "Human"
          description: "Versatile and adaptable."
          statMods: { str: 1, dex: 0, con: 0, int: 0, wis: 0, cha: 1 }
        ELF:
          displayName: "Elf"
          description: "Graceful and ancient."
          statMods: { str: -1, dex: 2, con: -2, int: 1, wis: 0, cha: 0 }
```

### Race Fields

| Field | Default | Notes |
|-------|---------|-------|
| `displayName` | (key name) | Shown during character creation. |
| `description` | `""` | Shown during character creation. |
| `statMods.str/dex/con/int/wis/cha` | `0` | Added to the character's base stat at creation. Can be negative. |

## Stats and Stat Bindings

### Stat Definitions

Path: `ambonmud.engine.stats.definitions`

Each key is the stat ID (uppercase). The six defaults are STR, DEX, CON, INT, WIS, CHA.
You can add custom stats but they must also be referenced in `statMods` (races),
equipment `stats` (world YAML), and status-effect `statMods`.

```yaml
ambonmud:
  engine:
    stats:
      definitions:
        STR:
          displayName: "Strength"
          abbreviation: "STR"
          description: "Physical power. Increases melee damage."
          baseStat: 10   # starting value for this stat at character creation
```

| Field | Default | Notes |
|-------|---------|-------|
| `displayName` | `""` | Full stat name shown on the score sheet. |
| `abbreviation` | `""` | Short label (e.g. "STR") used in the UI. |
| `description` | `""` | Tooltip text in the Creator and client. |
| `baseStat` | `10` | Starting value assigned to every new character for this stat, before race modifiers. |

### Stat Bindings

Path: `ambonmud.engine.stats.bindings`

Maps gameplay formulas to stat IDs.

```yaml
      bindings:
        meleeDamageStat: STR
        meleeDamageDivisor: 3     # (stat - 10) / divisor = extra damage per attack
        dodgeStat: DEX
        dodgePerPoint: 2          # dodge% per point above base
        maxDodgePercent: 30       # hard cap on dodge chance
        spellDamageStat: INT
        spellDamageDivisor: 3
        hpScalingStat: CON
        hpScalingDivisor: 5       # (stat - 10) / 5 * levels extra HP
        manaScalingStat: INT
        manaScalingDivisor: 5
        hpRegenStat: CON
        hpRegenMsPerPoint: 200    # ms shaved off regen interval per point above base
        manaRegenStat: WIS
        manaRegenMsPerPoint: 200
        xpBonusStat: CHA
        xpBonusPerPoint: 0.005    # 0.5% XP bonus per point above base
```

## Progression and Level Rewards

Path: `ambonmud.progression`

```yaml
ambonmud:
  progression:
    maxLevel: 50

    xp:
      baseXp: 100           # XP for level 2
      exponent: 2.0         # curve steepness (higher = faster ramp)
      linearXp: 0           # flat XP added per level (for shallower curves)
      multiplier: 1.0       # global XP gain multiplier
      defaultKillXp: 50     # fallback XP when a mob has no explicit reward

    rewards:
      hpPerLevel: 2         # base HP gained per level (before class bonus)
      manaPerLevel: 5       # base mana gained per level (before class bonus)
      fullHealOnLevelUp: true
      fullManaOnLevelUp: true
      baseHp: 10            # HP floor at level 1 (before per-level scaling)
      baseMana: 20          # Mana floor at level 1 (before per-level scaling)
```

### `rewards` Fields

| Field | Default | Notes |
|-------|---------|-------|
| `hpPerLevel` | `2` | Added to every class. A class with `hpPerLevel: 8` gets 8+2=10 HP per level. |
| `manaPerLevel` | `5` | Added to every class in the same way. |
| `fullHealOnLevelUp` | `true` | Restores full HP on level-up. |
| `fullManaOnLevelUp` | `true` | Restores full mana on level-up. |
| `baseHp` | `10` | The HP every character has at level 1, before CON scaling or class `hpPerLevel`. Raising this makes early levels less fragile. Must be >= 1. |
| `baseMana` | `20` | The mana floor at level 1. Raising this lets all classes cast more spells at low levels. Must be >= 0. |

**Formula for max HP at level N:**
```
maxHp = baseHp + (N-1) * (rewards.hpPerLevel + class.hpPerLevel)
      + floor((CON - 10) / hpScalingDivisor) * (N-1)
```

## Mob Tiers

Path: `ambonmud.engine.mob.tiers`

The four tiers control how mob stats scale with their `level` field in world YAML.
Individual mobs can override any field directly in the world YAML; tiers are the fallback.

```yaml
ambonmud:
  engine:
    mob:
      minActionDelayMillis: 8000    # minimum ms between mob actions
      maxActionDelayMillis: 20000

      tiers:
        weak:
          baseHp: 5
          hpPerLevel: 2
          baseMinDamage: 1
          baseMaxDamage: 2
          damagePerLevel: 0
          baseArmor: 0
          baseXpReward: 15
          xpRewardPerLevel: 5
          baseGoldMin: 1
          baseGoldMax: 3
          goldPerLevel: 1
        standard:
          baseHp: 10
          hpPerLevel: 3
          baseMinDamage: 1
          baseMaxDamage: 4
          damagePerLevel: 1
          baseArmor: 0
          baseXpReward: 30
          xpRewardPerLevel: 10
          baseGoldMin: 2
          baseGoldMax: 8
          goldPerLevel: 2
        elite:
          baseHp: 20
          hpPerLevel: 5
          baseMinDamage: 2
          baseMaxDamage: 6
          damagePerLevel: 1
          baseArmor: 1
          baseXpReward: 75
          xpRewardPerLevel: 20
          baseGoldMin: 10
          baseGoldMax: 25
          goldPerLevel: 5
        boss:
          baseHp: 50
          hpPerLevel: 10
          baseMinDamage: 3
          baseMaxDamage: 8
          damagePerLevel: 2
          baseArmor: 3
          baseXpReward: 200
          xpRewardPerLevel: 50
          baseGoldMin: 50
          baseGoldMax: 100
          goldPerLevel: 15
```

### Tier Formula

For a mob with `tier: standard` and `level: 5`, using `steps = level - 1 = 4`:

```
hp         = baseHp + steps * hpPerLevel        = 10 + 4*3 = 22
minDamage  = baseMinDamage + steps * damagePerLevel = 1 + 4*1 = 5
maxDamage  = baseMaxDamage + steps * damagePerLevel = 4 + 4*1 = 8
armor      = baseArmor (flat, not scaled)
xpReward   = baseXpReward + steps * xpRewardPerLevel = 30 + 4*10 = 70
goldMin    = baseGoldMin + steps * goldPerLevel  = 2 + 4*2 = 10
goldMax    = baseGoldMax + steps * goldPerLevel  = 8 + 4*2 = 16
```

Any of these can be overridden per-mob in world YAML:
```yaml
mobs:
  cave_troll:
    name: "Cave Troll"
    tier: elite
    level: 8
    hp: 200          # overrides tier formula entirely
    minDamage: 15
    maxDamage: 25
```

## Combat

Path: `ambonmud.engine.combat`

```yaml
ambonmud:
  engine:
    combat:
      maxCombatsPerTick: 20       # max simultaneous fight resolutions per tick
      tickMillis: 2000            # ms between combat rounds
      minDamage: 1                # global minimum damage floor
      maxDamage: 4                # global unarmed attack ceiling
      feedback:
        enabled: false            # show detailed roll breakdown to attacker
        roomBroadcastEnabled: false  # also show rolls to others in the room
```

**Note:** Class-specific threat multipliers are configured on each class definition (see
[Player Classes](#player-classes)) rather than here. Warriors have a default of 1.5;
all other classes default to 1.0.

## Regen

Path: `ambonmud.engine.regen`

```yaml
ambonmud:
  engine:
    regen:
      maxPlayersPerTick: 50         # max players regenerated per tick
      baseIntervalMillis: 5000      # ms between HP regen ticks at base CON
      minIntervalMillis: 1000       # fastest possible HP regen interval
      regenAmount: 1                # HP restored per regen tick

      mana:
        baseIntervalMillis: 3000    # ms between mana regen ticks at base WIS
        minIntervalMillis: 1000
        regenAmount: 1              # mana restored per regen tick
```

**Regen speed formula:** `interval = max(minInterval, baseInterval - CON_above_base * hpRegenMsPerPoint)`

## Abilities

Path: `ambonmud.engine.abilities.definitions`

Full ability reference is in `docs/WORLD_YAML_SPEC.md`. Quick summary of the fields a
balance designer most often adjusts:

| Field | Notes |
|-------|-------|
| `manaCost` | Mana consumed when cast. |
| `cooldownMs` | Milliseconds before the ability can be used again. |
| `levelRequired` | Minimum character level to use this ability. |
| `classRestriction` | Class ID that can use this ability. Omit for all classes. |
| `effect.type` | `DIRECT_DAMAGE`, `DIRECT_HEAL`, `APPLY_STATUS`, `AREA_DAMAGE`, `TAUNT`. |
| `effect.value` | Primary magnitude (damage, heal amount, or status stack count). |
| `effect.statusEffectId` | For `APPLY_STATUS` effects — must match a status effect key. |

## Status Effects

Path: `ambonmud.engine.statusEffects.definitions`

| Field | Notes |
|-------|-------|
| `effectType` | `DOT`, `HOT`, `STAT_BUFF`, `STAT_DEBUFF`, `STUN`, `ROOT`, `SHIELD`. |
| `durationMs` | Total effect duration in milliseconds. |
| `tickIntervalMs` | For DOT/HOT: ms between damage/heal ticks. |
| `tickValue` | Damage or heal per tick. |
| `statMods` | For STAT_BUFF/DEBUFF: map of stat ID to integer modifier (e.g. `STR: 3`). |
| `maxStacks` | Maximum stacks that can be applied simultaneously. |

## Crafting

Path: `ambonmud.engine.crafting`

```yaml
ambonmud:
  engine:
    crafting:
      maxSkillLevel: 10
      baseXpPerLevel: 100
      xpExponent: 1.5
      gatherCooldownMs: 30000      # ms between gathering attempts at the same node
      stationBonusQuantity: 1      # extra output items when crafting at a station
```
